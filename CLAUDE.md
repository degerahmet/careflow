## Development environment

Everything runs inside Docker. There is no local Python or Node setup — all commands go through `docker compose exec`.

```bash
# First-time setup
cp .env.example .env
docker compose up --build

# Run migrations (after first build or after adding new models)
docker compose exec backend python manage.py migrate

# Rebuild a single service after dependency changes
docker compose up --build backend
docker compose up --build frontend
```

## Common backend commands

```bash
# Django management
docker compose exec backend python manage.py <command>
docker compose exec backend python manage.py createsuperuser
docker compose exec backend python manage.py shell

# Validate the OpenAPI schema (catches broken annotations)
docker compose exec backend python manage.py spectacular --validate

# Create a new Django app inside the container
docker compose exec backend python manage.py startapp <appname>
```

## Architecture

### Monorepo layout
- `backend/` — Django 5 + DRF 3.15, Django project is named `config` (settings, urls, wsgi live in `backend/config/`)
- `frontend/` — Vue 3 + Vite, proxies `/api/*` to the backend container via `vite.config.js`
- `prompts/` — AI prompt logs organised

### Request flow
Browser → Vite dev server (`:5173`) → proxy `/api/*` → Django (`:8000`) → PostgreSQL (`:5432`)

The Vite proxy target is `http://backend:8000` (Docker service name), so it only works inside Docker networking.

### Backend conventions
- All config is read from environment variables via `python-decouple`; no hardcoded values in settings
- New Django apps should be registered in `INSTALLED_APPS` in `backend/config/settings.py`
- New API viewsets are registered on the `DefaultRouter` in `backend/config/urls.py`
- CORS is open (`CORS_ALLOW_ALL_ORIGINS = True`) only when `DJANGO_DEBUG=True`
- DRF uses JSON-only renderers/parsers by default; no browsable API

### API documentation
Swagger UI and ReDoc are auto-generated by `drf-spectacular` from registered viewsets — no manual schema maintenance needed.

| UI         | URL                               |
|------------|-----------------------------------|
| Swagger UI | http://localhost:8000/api/docs/   |
| ReDoc      | http://localhost:8000/api/redoc/  |
| Raw schema | http://localhost:8000/api/schema/ |

Use `@extend_schema` decorator from `drf_spectacular.utils` when adding viewsets that need richer documentation.

### Environment variables
See `.env.example` for all required variables. Key ones:
- `DJANGO_SECRET_KEY`, `DJANGO_DEBUG`
- `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`
- `VITE_API_BASE_URL` (used by the frontend at build time)
