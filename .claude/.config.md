# Project Context

This is a take-home assignment for an AI-assisted appointment booking system.

Roles:
- member
- provider
- admin

Tech Stack:
- Backend: Django + Django REST Framework
- Frontend: Vue.js (Vite)
- Database: PostgreSQL
- Infrastructure: Docker Compose

---

# Development Principles

- This is a 2-day MVP project
- Prioritize simplicity and clarity over completeness
- Avoid over-engineering
- Write clean, readable, interview-friendly code
- Prefer explicit implementations over abstractions

---

# Architecture Guidelines

- Use a monorepo structure:
  - /backend
  - /frontend
  - /prompts

- Backend should be modular and organized by domain
- Keep Django apps simple and focused
- Do not introduce unnecessary services (no microservices, no event bus)

---

# API Design Rules

- Use RESTful conventions
- Keep endpoints predictable and simple
- Use clear naming
- Return structured JSON responses

---

# AI Usage Constraint

- AI should assist the booking flow, not fully automate it
- AI extracts structured data from natural language
- User must always confirm final actions

---

# Prompt Tracking Requirement

- All important prompts must be stored under /prompts
- Prompts should be reusable and clearly named
- Do not ignore this requirement

---

# Coding Constraints

- Do not implement unnecessary features
- Do not add production-level complexity
- Do not introduce tools not explicitly needed
- Always prefer the simplest working solution

---

# Workflow Rule

Before making changes:
1. summarize the task
2. list assumptions
3. propose a minimal plan
4. list files to change

Wait for confirmation before implementing unless explicitly told to proceed.
